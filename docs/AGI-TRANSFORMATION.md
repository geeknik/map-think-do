# Architecture Notes

This repository uses "cognitive", "persona", and "metacognitive" terminology to describe how it organizes reasoning heuristics. Those names reflect implementation style and prompt framing, not a claim of sentience or AGI.

## Current Architecture

The main server lives in [`src/server.ts`](/home/geeknik/dev/map-think-do/src/server.ts) and exposes the `map-think-do` MCP tool.

Important subsystems:

- [`src/cognitive/cognitive-orchestrator.ts`](/home/geeknik/dev/map-think-do/src/cognitive/cognitive-orchestrator.ts): coordinates plugin execution, state updates, recommendations, and memory writes.
- [`src/cognitive/state-tracker.ts`](/home/geeknik/dev/map-think-do/src/cognitive/state-tracker.ts): tracks session and cognitive-state heuristics.
- [`src/state/`](/home/geeknik/dev/map-think-do/src/state): maintains unified application state and adapter-driven synchronization.
- [`src/memory/`](/home/geeknik/dev/map-think-do/src/memory): persistent storage abstractions and implementations.
- [`src/prompts/`](/home/geeknik/dev/map-think-do/src/prompts): prompt templates and prompt value persistence.

## Reasoning Model

The server supports:

- linear thought sequences
- revisions of earlier thoughts
- branches for alternative approaches
- bias detection heuristics
- intervention and recommendation generation
- persisted memory for later retrieval and scoring

The numeric outputs such as `metacognitive_awareness` and `breakthrough_likelihood` are implementation-defined heuristics generated by this codebase.

## Limits

The current implementation does not establish:

- real consciousness
- general intelligence
- autonomous agency
- guaranteed correctness of recommendations

It is best understood as an MCP reasoning server with richer state and metadata than a simple sequential-thought tool.

## Ongoing Documentation Rule

Documentation in this repository should describe observable behavior, supported interfaces, and known limits. Marketing language that implies capabilities beyond the implementation should be treated as stale and corrected.
